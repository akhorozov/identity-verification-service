This commit delivers the complete implementation of T2 (Core Domain Models),

CHANGES:

1. Core Domain Models (src/AddressValidation.Api/Domain/)
   - AddressInput.cs: Request model with cross-field validation
     * Validates: Street, City+State OR ZipCode
     * Supports: Standard addresses, ZIP+4 format
     * Uses: IValidatableObject for complex business rules

   - ValidatedAddress.cs: USPS CASS-certified standardized address
     * Includes: DeliveryLine1, LastLine, USPS components
     * Nullable fields: Support failed validation responses
     * Parseable: PrimaryNumber, StreetName, StreetSuffix, etc.

   - AddressAnalysis.cs: USPS DPV analysis and metadata
     * DPV Indicators: MatchCode, Footnotes, CMRA, Vacant, Active
     * Postal Flags: ResidentialDeliveryIndicator, PoBoxIndicator
     * Sequencing: EnhancedLineOfTravel, SuiteLinkMatch

   - GeocodingResult.cs: Geographic coordinates (WGS84)
     * Fields: Latitude, Longitude, Precision, CoordinateLicense
     * Optional: Null if geocoding unavailable

   - ValidationMetadata.cs: Response context metadata
     * Tracking: ProviderName, ValidatedAt, CacheSource, ApiVersion
     * Performance: RequestDurationMs
     * Tracing: CorrelationId

   - ValidationResponse.cs: Aggregate response model
     * Combines: InputAddress, ValidatedAddress, Analysis, Geocoding, Metadata
     * Status: validated, ambiguous, invalid, undeliverable
     * Batch Support: InputIndex for ordering

   - AddressHashExtensions.cs: Deterministic hashing and cache keys
     * ComputeHash: SHA-256 of normalized address (case-insensitive)
     * GenerateCacheKey: Format addr:v1:{hash} with versioning
     * ExtractHashFromCacheKey: Parse hash from cache key
     * IsValidCacheKey: Validate key format
